## 1. 六棱镜探索

### 1.1 产品镜

| 问题 | 回答 |
|------|------|
| 用户原始诉求 | 减少误解需求、漏功能、关任务时无法证明「做对了」 |
| In Scope 对应 | M4 文档体系延伸；记忆层 Epic F 增强 |
| Out of Scope 边界 | 不实现 AI 自动读心；人工/Agent 仍须填 intake |
| 成功指标 | 每任务有 intake+audit；八项逻辑检查可追踪 |

### 1.2 用户镜

| 问题 | 回答 |
|------|------|
| 受益者 | Skill 维护者、接任务的 Agent、Code Review 者 |
| 体验差异 | 前：关任务只有 diff；后：需求↔实现三角验证 |
| Happy path | init-intake → 填六棱镜 → 实施 → 生成 audit 通过 |
| 异常 path | 无 intake → audit 警告；逻辑项未勾选 → 不可关任务 |

### 1.3 功能镜

| 问题 | 回答 |
|------|------|
| 涉及模块 | memory-layer；generate-memory-report.py |
| 与 14 模块关系 | 无关（纯文档/工具链） |
| 输入 / 输出 | 输入：task-id、intake；输出：audit.md |
| 关联 spec | docs/memory/README.md；无 scan spec 变更 |

### 1.4 技术镜

| 问题 | 回答 |
|------|------|
| 架构层级 | 记忆层之上加「需求层」 |
| module-map 模块 | memory-layer 扩展 |
| 是否改 action 链路 | 否 |
| 配置加载 | requirement-lens.yaml |

### 1.5 边界镜

| 问题 | 回答 |
|------|------|
| 空项目 / 无语言 | 不适用；纯文档任务 |
| 工具或 TG 失败 | 脚本无 pyyaml 时自动 pip install |
| fail-on-findings | 不影响 |
| fork PR / Secrets | 不影响 |

### 1.6 集成镜

| 问题 | 回答 |
|------|------|
| 业务仓 workflow | 无影响 |
| 破坏现有接入？ | 否 |
| 制品 / TG | 无影响 |
| 需同步的文档 | memory/README、REQUIREMENT-GUIDE、CONTEXT、tasks Epic G |
